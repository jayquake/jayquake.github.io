Buttons should be tagged for assistive technology

If interactive elements cannot be identified as buttons, screen reader users may not realize the element is actionable, which can stop them from submitting forms, opening dialogs, or performing other intended actions.

Compliant
11 examples

Add role="button" to the custom interactive element, or use a HTML <button> element.

1. Follow proper HTML semantics

2. Ensure screen reader compatibility

3. Test with assistive technologies
