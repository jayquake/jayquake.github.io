CAPTCHA used in authentication should not involve cognitive function tests

With the exception of challenges that require recognition of objects that are generally familiar to users, CAPTCHA used during authentication should not require users to perform cognitive function tests, such as recalling site-specific passwords or solving a puzzle.

Needs Fix
2 examples

Provide an alternative authentication method that avoids cognitive tasks, such as a two-factor verification step. If CAPTCHA is required, use services that offer strong accessibility support, such as Google reCAPTCHA v3.

1. Review the HTML structure

2. Apply proper accessibility attributes

3. Test with screen readers
