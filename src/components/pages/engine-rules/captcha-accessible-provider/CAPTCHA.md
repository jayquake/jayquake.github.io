## Accessible Captchas

The following list details accessible CAPTCHA solutions in order of their usage and provides brief insights into their compliance with the Web Content Accessibility Guidelines (WCAG):

### Google reCaptcha

Google reCaptcha is one of the most widely used CAPTCHA solutions. More information can be found here: [Google reCaptcha](https://support.google.com/recaptcha/answer/6175971?hl=en).

### Cloudflare Turnstile

Cloudflare Turnstile is WCAG compliant but has known issues. More details can be accessed here: [Cloudflare Turnstile FAQ](https://developers.cloudflare.com/turnstile/frequently-asked-questions/#does-turnstile-conform-to-wcag-2.1-level-aa-accessibility-standard).

### hCaptcha

hCaptcha is WCAG compliant but requires registration, specific settings, and periodic logins. Further information is available here: [hCaptcha Accessibility](https://www.hcaptcha.com/accessibility).

### AWS WAF Captcha

AWS WAF Captcha uses a modal which is hard to detect. More details can be found here: [AWS WAF Captcha](https://aws.amazon.com/about-aws/whats-new/2022/06/aws-waf-captcha-generally-available/).

### GeeTest Captcha

GeeTest Captcha is WCAG compliant, but clients need to activate the accessibility feature. It uses custom element IDs making it harder to match. More information can be found here: [GeeTest Accessibility](https://docs.geetest.com/BehaviorVerification/Accessibility/accessibility).

### Friendly Captcha

Friendly Captcha is WCAG compliant. More information can be found here: [Friendly Captcha Accessibility](https://friendlycaptcha.com/insights/captcha-accessibility/).
